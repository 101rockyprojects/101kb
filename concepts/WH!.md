# Concepts

This folder contains evergreen engineering notes: definitions, mental models, and patterns that show up across projects.

## What belongs here

- System design: architectures, communication patterns, tradeoffs.
- Programming concepts: paradigms, principles, language-agnostic ideas.
- Terminology: short, clear definitions you can link to from anywhere.

## Conventions

- One idea per note. If a note starts to become a "bag of things", split it.
- Start with a 1-3 sentence definition, then add sections like `Why it matters`, `Examples`, and `Pitfalls`.
- Link aggressively using `[[...]]`:
  - Prefer `[[concepts/...]]` when there is name ambiguity.
  - Add a small `Related` section at the bottom of notes.
